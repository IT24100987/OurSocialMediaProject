🔐 BACKEND FULL REQUIREMENTS
==========================

SERVER (server.js):
- Express server on port 5000
- CORS enabled for http://localhost:5173
- JSON body parser with limit 10mb
- Connect MongoDB via db.js
- Mount all routes under /api/

AUTH CONTROLLER:
- POST /api/auth/register
  → hash password with bcrypt (salt 10)
  → create User in DB
  → return JWT token + user info (name, email, role)
- POST /api/auth/login
  → find user by email
  → compare password with bcrypt
  → return JWT token + user info

JWT MIDDLEWARE (authMiddleware.js):
- Extract Bearer token from Authorization header
- Verify token with JWT_SECRET
- Attach decoded user to req.user
- Return 401 if missing or invalid

ROLE MIDDLEWARE (roleMiddleware.js):
- Accept array of allowed roles
- Check req.user.role against allowed roles
- Return 403 if not authorized

USER CONTROLLER (Full CRUD):
- GET /api/users → all users (Admin only)
- GET /api/users/:id → single user (Admin only)
- POST /api/users → create user (Admin only)
  → include AI role recommendation in response
- PUT /api/users/:id → update user or role (Admin only)
- DELETE /api/users/:id → deactivate user (Admin only)

CLIENT CONTROLLER (Full CRUD):
- GET /api/clients → all clients with search & filter by package/status
- GET /api/clients/:id → single client profile
- POST /api/clients → create client (Admin/Manager)
- PUT /api/clients/:id → update client info or package
- DELETE /api/clients/:id → deactivate client
- POST /api/clients/register → public route for landing page registration
  → creates Client record with registeredFrom: 'landing_page'
  → also creates User account with role 'Client'
  → sends back JWT so client is auto-logged in

FEEDBACK CONTROLLER (Full CRUD):
- POST /api/feedback → submit feedback
  → auto-calculate sentiment using AI
  → store sentiment + AI suggestion in DB
- GET /api/feedback → all feedback (Admin/Manager)
- GET /api/feedback/my → logged-in client's feedback only
- PUT /api/feedback/:id → edit (Client owner only, within 24 hours)
- DELETE /api/feedback/:id → delete (Admin or owner)

TASK CONTROLLER (Full CRUD):
- GET /api/tasks → all tasks (Admin/Manager)
- GET /api/tasks/my → tasks assigned to logged-in Staff
- GET /api/tasks/calendar/:month → tasks for a given month YYYY-MM
- POST /api/tasks → create task (Admin/Manager)
- PUT /api/tasks/:id → update task details or status
- DELETE /api/tasks/:id → delete task (Admin/Manager)

PAYMENT CONTROLLER (Full CRUD):
- GET /api/payments → all payments (Admin/Manager)
- GET /api/payments/stats → total revenue, this month revenue, pending count, overdue count
- GET /api/payments/client/:clientId → payments for one client
- POST /api/payments → record new payment
  → auto-generate invoiceNumber as INV-YYYY-XXXX
- PUT /api/payments/:id → update payment status or details
- DELETE /api/payments/:id → delete record (Admin only)

ANALYTICS CONTROLLER (Full CRUD):
- GET /api/analytics → all records (Admin/Manager)
- GET /api/analytics/summary → totals for reach, impressions, engagement
- GET /api/analytics/client/:clientId → analytics for one client
- GET /api/analytics/monthly/:month → monthly report YYYY-MM
- POST /api/analytics → add analytics record
- PUT /api/analytics/:id → update record
- DELETE /api/analytics/:id → delete record

AI CONTROLLER (aiController.js) — use Anthropic Claude API for all:
- POST /api/ai/chat → general AI chat for any question about the system
- POST /api/ai/sentiment → analyze feedback text, return positive/neutral/negative
- POST /api/ai/role-recommend → suggest best role based on user description
- POST /api/ai/permission-explain → explain what a given role can access
- POST /api/ai/trends → detect recurring issues from array of feedback texts
- POST /api/ai/suggestions → generate campaign improvement recommendations
- POST /api/ai/monthly-summary → produce monthly client satisfaction summary
- POST /api/ai/analytics-insight → analyze analytics data and give insights

AI HELPER (utils/aiHelper.js):
Create a reusable function callClaude(prompt) that:
- Calls https://api.anthropic.com/v1/messages
- Uses model: claude-opus-4-5
- Uses max_tokens: 1024
- Returns the text response
- Has try/catch with proper error message
Use this function in all AI controllers.
